# Atlas Endgame — Game Design Document

## Overview

Игра сразу начинается с атласов — нет кампании или предыстории. Первый атлас служит обучением.

Gameplay построен вокруг **Atlas** — генерируемой сетки карт, которую игрок крафтит, проходит и сбрасывает. Внутри атласа gameplay работает как roguelite-ран с фиксированным деревом пассивок и рандомным скиллом. Между ранами игрок перестраивает дерево и крафтит следующий атлас.

## Core Loop

```
┌─────────────────────────────────────────────────┐
│                     META                        │
│                                                 │
│   Перестройка дерева пассивок                   │
│   Крафт нового атласа                           │
│   ↓                                             │
│ ┌─────────────────────────────────────────────┐ │
│ │                  RAN (Atlas)                │ │
│ │                                             │ │
│ │   Получить рандомный скилл                  │ │
│ │   Проходить карты, прокачиваться по дереву  │ │
│ │   Адаптировать скилл через апгрейды (1/3)   │ │
│ │   Clear / speedrun POI → завершить атлас    │ │
│ └─────────────────────────────────────────────┘ │
│   ↓                                             │
│   Получить валюту и ноды                        │
│   Вернуться к META                              │
└─────────────────────────────────────────────────┘
```

## Atlas

### Структура
- Сетка клеток. Минимальный размер 5×5, увеличивается через крафт атласа.
- Игрок стартует в центре.
- На сетке размещены **Points of Interest (POI)** — боссы, уникальные награды, особые механики.
- Один из POI — **главный босс атласа**.

### Крафт атласа (до генерации)
Перед генерацией игрок тратит ресурсы чтобы влиять на:
- Количество и тип POI
- Типы карт (биомы, лейауты)
- Сложность и награды (risk/reward)
- Размер сетки (от 5×5 и выше)
- Модификаторы на весь атлас

### Навигация
- Перемещение только в **соседние клетки** (4 направления) от уже зачищенных.
- Фронт расширяется от центра — игрок прокладывает маршрут.
- Вся сетка **полностью видна** с самого начала (тип клетки, сложность, награды, POI).

### Сложность и прокачка
- Сложность определяется **порядком прохождения**: 1-я зачищенная клетка = уровень 1, 2-я = уровень 2 и т.д.
- В атласе 5×5 — 25 уровней сложности. Расположение на сетке не влияет.
- Босс — одна из клеток, его сложность растёт наравне с остальными.
- За каждую зачищенную клетку — **1 очко дерева пассивок**.
- **Ключевая дилемма:** фарми карты чтобы стать сильнее, но босс тоже растёт. Нужно найти sweet spot — достаточно прокачаться, но не перекачать босса.

### Смерть
- Смерть на любой карте = **конец рана**. Весь атлас потерян.
- Классический roguelite подход — каждая карта имеет stakes.

### Завершение атласа
- **Цель рана — убить главного босса атласа.**
- Босс **виден на сетке с самого начала** — игрок планирует маршрут.
- Игрок сам решает когда идти на босса — можно сразу (если уверен), можно подфармить.
- Убийство босса = **ран завершён**, награды получены, атлас закрывается.

### Магазин
- Текущий вариант: магазин доступен **после каждой карты** (орбы, артефакты, прокачка по дереву).
- В перспективе: магазин только на **POI**, между обычными картами — только выбор 1 из 3 апгрейдов.

### Карта (клетка)
- Каждая клетка — **один энкаунтер** (один бой). Короткие, быстрые.
- Атлас = много коротких боёв с прогрессией между ними.

### Прогресс в ране
- Очки дерева, апгрейды скилла, артефакты — всё **сбрасывается** после завершения атласа (успех или смерть).
- Каждый ран начинается с нуля. Чистый roguelite.
- Между ранами сохраняется только: мета-валюта, новые ноды, структура дерева.
- Нет персистентного уровня/опыта — прогрессия только через дерево.

## Дерево пассивок

### Концепция
Большое дерево нод (по типу PoE passive tree), но игрок **сам его строит**, а не навигирует фиксированное.

### Два режима работы

**В мете (между ранами) — конструктор:**
- Двигать ноды
- Менять ноды (тип, значения)
- Добавлять / удалять ноды
- Создавать / удалять связи между нодами
- Вставлять гемы в сокеты
- Respec полностью бесплатный

**В ране (внутри атласа) — фиксированное дерево:**
- Дерево заблокировано от изменений
- Игрок получает очки за прокачку (level up, награды карт)
- Тратит очки на ноды по путям которые сам же построил

### Прогрессия размера
- В начале игры дерево маленькое (~15 нод)
- Игрок вручную увеличивает его по мере прогрессии
- Ветеран: 200+ нод, сложные multi-path структуры, gem sockets

### Ограничения
- Максимум связей на ноду (предотвращает "связать всё со всем")
- Ноды и связи стоят ресурсы для добавления

### Получение нод
- **Обычные ноды:** крафт через валюту, получаемую за карты
- **Уникальные / имбовые ноды:** награда за POI или победу над главным боссом атласа

## Скилл в ране

### Выдача
- В начале рана игрок выбирает **1 скилл из 3 рандомных**.
- Дополнительные скиллы можно получить за **POI** по ходу рана.

## Экономика

### Валюты (TBD)
- **Основная валюта** — за карты. Тратится на: крафт нод, крафт атласа, связи в дереве.
- **Редкая валюта** — за POI / боссов. Тратится на: уникальные ноды, мощные крафты атласа.

### Reward loop
```
Карты → валюта → ноды + крафт атласа → лучшие раны → больше валюты
POI / боссы → уникальные ноды → сильнее дерево → сложнее атласы → лучшие POI
```

## Open Questions

- Форма атласа: всегда квадрат или варьируется?
- Мета-прогрессия: как разблокируются новые опции крафта атласа?
- Типы POI кроме главного босса
- Содержание нод дерева пассивок
- Адаптация скилла и взаимодействие с деревом
- Валюты: сколько, какие, как разделяются между мета и раном
- Апгрейды после карты (1 из 3): что именно предлагается
- Multiplayer в будущем?
