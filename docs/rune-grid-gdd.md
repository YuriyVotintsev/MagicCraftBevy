# Rune Grid — Game Design Document

## Концепция

Гексагональная сетка (4 клетки вдоль стороны, 37 ячеек макс.) — центральная система прогрессии внутри run. Руны дают **базовый эффект** (слабый сам по себе) и **позиционный эффект** — читают состояние соседей и/или пишут в них модификаторы. Правильное позиционирование создаёт синергии, превращая слабые руны в мощные комбинации.

## Жанр: переход от incremental к roguelite

Этот документ фиксирует **смену жанра** проекта с incremental arena shooter на **roguelite / survivor-like** (ориентиры: Brotato, Vampire Survivors, 20 Minutes Till Dawn).

**Что меняется по сравнению со старым incremental-дизайном:**

| Аспект | Incremental (старый) | Roguelite (новый) |
|--------|---------------------|-------------------|
| Прогресс | Копится между run (skill tree persist) | Сбрасывается каждый run (сетка+руны обнуляются) |
| Смерть | Конец ресурса, билд сохраняется | Permadeath: полный reset билда |
| Run | Бесконечная петля wave→shop→wave | Законченный забег с целью (Wave N / босс) |
| Выбор | Свободный магазин, всё доступно | Random offer 3-4 руны, lock-in |
| Вариативность | Количественная (те же числа быстрее) | Качественная (синергии меняют механику) |
| Skill-check | Минимальный (numbers-go-up) | Позиционка, dodge, aura-пазл в реальном времени |

**Roguelite-маркеры в дизайне сетки:**
- **Run как единица опыта** — забег 15-40 мин с кульминацией, а не бесконечная петля.
- **Permadeath с полным reset** — при смерти теряется вся сетка, все руны, все разблокированные ячейки.
- **Random offer каждый Shop** — 3-4 случайные руны из пула, нельзя просто «докупить нужное».
- **Синергии меняют механику** — Echo удваивает Write-эффекты, Chain передаёт значение по направлению, Read в одной геометрии + Write в другой создаёт нелокальные связи. Не просто +числа.
- **Джокеры (6 слотов)** — внешние уникальные руны, меняющие правила, задают форму билда (аналог джокеров Balatro).
- **Мета-прогрессия между run** — разблокировка новых рун и джокеров в общий пул (расширяет варианты, не усиливает текущий билд).

## Модель волн

- Run — **целевой забег до Wave N** (или до финального босса, TBD), а не бесконечная петля.
- Каждая волна имеет **фиксированное количество врагов**
- Враги спавнятся поштучно (не все сразу)
- Когда все враги волны заспавнены — ждём, пока всех убьют
- После уничтожения всех врагов → Shop фаза → следующая волна
- **Нет** плавного уменьшения жизней. Жизни теряются только от урона врагов
- **Game Over** при 0 HP — permadeath: полный сброс **всего билда** (сетка, руны, джокеры, разблокированные ячейки, gold). Остаётся только мета-прогрессия (пул доступных рун и джокеров).

## Гексагональная сетка

Сетка — правильный гексагон с 4 клетками вдоль каждой стороны (side=4, radius=3).

```
Полная сетка (37 ячеек):

        ○ ○ ○ ○           ← edge row (4 cells)
       ○ ○ ○ ○ ○          ← 5 cells
      ○ ○ ○ ○ ○ ○         ← 6 cells
     ○ ○ ○ ○ ○ ○ ○        ← center row (7 cells)
      ○ ○ ○ ○ ○ ○         ← 6 cells
       ○ ○ ○ ○ ○          ← 5 cells
        ○ ○ ○ ○           ← edge row (4 cells)

Ring 0 (center):  1 cell
Ring 1:           6 cells   (total: 7)
Ring 2:          12 cells   (total: 19)
Ring 3:          18 cells   (total: 37)
```

Координатная система — axial (q, r). Центр = (0, 0).

### Прогрессия размера

Начинаем с внутренних колец, разблокируем ячейки за gold:
- **Старт:** Ring 0-1 (7 ячеек) — компактно, легко понять механику
- **Покупка:** отдельные ячейки Ring 2 и Ring 3 за фиксированную цену (Ring 2 дешевле Ring 3)
- **Макс:** все 37 ячеек

### Общий лимит рун

Количество рун на сетке жёстко ограничено и **автоматически выводится из числа разблокированных ячеек**:

```
rune_limit = ceil(unlocked_cells / 2)
```

Примеры: 7 ячеек → 4 руны, 19 → 10, 37 → 19. Ровно половина разблокированных ячеек всегда пуста.

**Зачем:** пустые ячейки — не «мёртвый вес», а **ресурс для Void/Isolate-синергий** (см. ниже). Привязка лимита к ячейкам убирает отдельную покупку, оставляет один осмысленный выбор за gold — разблокировать ячейку (и двигает лимит, и открывает место под синергию).

Ratio `1/2` — стартовое значение для балансного тюнинга, не догма.

### Пустые ячейки как цели синергий

Некоторые руны читают **пустые разблокированные ячейки** в своём паттерне и получают от них бонус. Это делает нетривиальным вопрос «куда ставить руну» даже при избытке ячеек и превращает «свободное место» в активный элемент билда.

Примеры (конкретные руны вынесены в таблицы ниже):
- *Void*-архетип: +X% crit за каждую пустую ячейку в Ring.
- *Echo Chamber*: удваивает свой Write-эффект, если все 6 Adjacent-ячеек пусты.
- *Hermit*: работает только если Adjacent полностью пуст, но даёт большой base effect.

**Балансный инвариант:** доля void/isolate-рун в пуле должна быть достаточно высокой, чтобы типичная сетка естественно разбивалась на **смесь кластеров и одиночек**, а не деградировала в «комок соседских синергий в одной половине + несколько изолянтов на отшибе». Если void-руны редки, игрок будет всегда кучковать — пустая половина сетки превратится в мёртвую зону. Конкретная доля — параметр балансировки (ориентир: ~30-40% пула должны любить пустоту или изоляцию).

### 6 направлений (hex neighbors)

```
    NW  NE
  W  ●  E
    SW  SE
```

Каждая ячейка имеет до 6 соседей. Это задаёт геометрию всех паттернов.

## Анатомия руны

Каждая руна имеет:

| Свойство | Описание |
|----------|----------|
| **Base Effect** | Прямой бонус к стату игрока (слабый) |
| **Read** | Опциональное: `(Pattern, Filter, Formula)` — что и откуда руна считывает с сетки; даёт скаляр/булев |
| **Write** | Опциональное: `(Pattern, Filter, Effect)` — куда руна пишет результат; `Pattern` может быть `Self`. Если пуст — руна существует ради своего Base Effect, её цена — занимаемый слот |
| **Tier** | Common, Rare, Epic, Legendary (масштаб эффектов). Все тиры могут появляться с самого начала |
| **Limit** | Макс. копий на поле. Legendary=1, некоторые Common=∞ |

Модель руны — строго **одна** пара `Read? → Write?`. Оба слота опциональны. Параметры Effect в Write могут использовать значение из Read.

- **Self-архетип** — Write с `Pattern = Self`, обычно в паре с Read по окружению (собирает данные → модифицирует себя).
- **Other-архетип** — Write с любым другим паттерном (пишет в цели). Read может быть, а может и не быть.
- **Cross-pattern** — Read по одной геометрии, Write в другую. Поддерживается штатно.
- **Pure-base** — ни Read, ни Write. Руна существует ради Base Effect; цена — занимаемый слот. Обычно редкий тир: за бесплатную статичку с высоким base приходится платить редкостью в пуле.

Более сложные рулбук-моды, conditional-триггеры и сложные event-реакции — это **джокеры**, не руны. Руны остаются линейной моделью Read → Write.

## Паттерны — Hex

Паттерны адаптированы под гексагональную геометрию:

```
Adjacent (6 соседей)        Line (одна из 3 осей)
                            
     X X                         X
    X * X                       X
     X X                    X X * X X
                                X
                                 X

Ring (12 клеток dist=2)     Cone (треугольник в одном направлении)

    X X X                        X X
   X . . X                     X X X
  X . * . X                   X X * 
   X . . X
    X X X

Half (полусфера)            All Same Rune
                            (anywhere on grid)
   X X X X                  
  X X X X                   S . . .
   X X *                     . S . .
    . .                     . . . S
     .
```

### Полный список паттернов

| Паттерн | Hex-описание | Кол-во целей (макс.) | `include_self` по умолчанию |
|---------|-------------|---------------------|----------|
| **Adjacent** | 6 соседних ячеек | 6 | `false` |
| **Line** | Все клетки по одной из 3 осей (через центр руны) | до 6 | `false` |
| **Ray** | Луч в одном направлении (E, NE, NW, W, SW, SE) | до 3 | `false` |
| **Ring** | Все клетки на расстоянии ровно 2 | 12 | `false` |
| **Cone** | Треугольник в направлении (расширяется) | до 6 | `false` |
| **Half** | Все клетки по одну сторону от руны | до ~18 | `false` |
| **Nearby (dist ≤ 2)** | Все в радиусе 2 (Adjacent + Ring) | 18 | `false` |
| **All Same Rune** | Все копии той же руны на сетке | variable | `true` |
| **All on Grid** | Все руны на сетке | variable | `true` |
| **Self** | Сама руна (применимо и для Read, и для Write) | 1 | `true` |

### Параметр `include_self`

У геометрических и count-подобных паттернов есть булев параметр `include_self`, определяющий, учитывается ли **сама руна-источник** в зоне паттерна. Дефолт выбран так, как ощущается естественно: геометрические паттерны описывают «окружение», поэтому сам центр в них не входит; «все руны» — по смыслу включают всех, включая себя.

Паттерн в сигнатуре Read/Write можно писать с override'ом параметра:
- `(Adjacent, any, count)` — 6 соседей, без центра (дефолт).
- `(Adjacent{include_self: true}, any, count)` — 7 ячеек: 6 соседей + центр.
- `(AllOnGrid{include_self: false}, rune, count)` — все руны кроме самой.

Для `Self` параметр формально применим (дефолт `true` — пишем в себя), но `Self{include_self: false}` вырождается в «никто» — паттерн без целей. Не ошибка модели, а легитимный способ выключить Write (хотя проще просто не указывать Write).

## Модель эффекта (Read / Write)

Эффект руны — это пара:

```
Read(Pattern, Filter, Formula) → value   [опционально]
Write(Pattern, Filter, Effect(value))    [опционально]
```

Каждый слот — типизированный, строго перечисленных вариантов. Это позволяет UI автоматически извлечь все задействованные паттерны и подсветить их при hover/drag (без парсинга free-form строк).

### Pattern

Список паттернов из секции «Паттерны — Hex», включая `Self` (применим и в Read, и в Write; означает «сама руна»).

### Filter

Фильтрует, какие ячейки/руны в Pattern учитываются.

| Filter | Значение |
|--------|----------|
| `any` | Любая ячейка (дефолт) |
| `empty` | Только пустые разблокированные ячейки |
| `rune` | Только ячейки с рунами |
| `tier(T)` | Только руны тира T (Common/Rare/Epic/Legendary) |
| `name(N)` | Только руны с конкретным именем (для синергий типа «все Spark») |

### Formula (для Read)

Простая функция над отфильтрованными ячейками, возвращает скаляр или булев.

| Formula | Что возвращает |
|---------|----------------|
| `count` | Количество ячеек, прошедших Filter |
| `sum(stat)` | Сумма конкретного стата base у отобранных рун |
| `max(stat)` | Максимум конкретного стата base |
| `any` | Булев: есть ли хотя бы одна подходящая ячейка |
| `none` | Булев: пусто ли |

Результат Read подставляется в параметры Effect в Write как переменная `N` (или `V` для value-типов).

### Effect (для Write)

Что именно делаем в целях Write.

| Effect | Что делает |
|--------|------------|
| `Grant(value)` | Добавляет плоский бонус (к base или к стату) |
| `Amplify(factor)` | Умножает base целей |
| `Suppress(factor)` | Уменьшает base целей |
| `Echo` | Эффекты целей срабатывают дважды (рекурсия макс. 1 уровень) |
| `Chain(value)` | Передаёт значение дальше по Ray (цепочка) |

Значение `value` в Grant / Chain / Suppress / Amplify может быть:
- Константа (`+3%`, `x1.2`).
- Выражение от `N` из Read (`N * 0.5%`, `10% if N == 0 else 0`).
- `own.base` — собственный base руны.

## Пример синергии (Hex)

```
Фрагмент сетки:

      [Lens] [Spark]
       [Spark]  [ ... ]

Lens (Rare):
  Base:  +1% all stats
  Read:  —
  Write: (Adjacent, any, Amplify(x1.2))

Spark (Common):
  Base:  +3% damage
  Read:  —
  Write: (Adjacent, any, Grant(+2% damage))

Оба Spark — соседи Lens и соседи друг друга.

Spark #1 получает:
  Base:                       +3% dmg
  от Spark #2 (Write):        +2% dmg (Adjacent Grant)
  Lens (Write Amplify x1.2):  усиливает Write Spark #1
  Итого Spark #1 base:        3% + 2% = 5% dmg
  Write Spark #1 после Lens:  раздаёт +2.4% вместо +2%

Spark #2 получает:
  Base:                       +3% dmg
  от Spark #1 (Write):        +2.4% dmg (усилен Lens)
  Итого:                      5.4% dmg

vs +3% без синергий у каждого
```

## Примеры конкретных рун

Формат записи: `Руна | Base | Read | Write | Limit`. Оба слота (Read, Write) опциональны — `—` означает, что слот не заполнен. Пустой Read — руна пишет по константе. Пустой Write — pure-base (см. пример Nugget). Руна с обоими пустыми слотами легальна, её цена — занимаемый слот сетки.

### Common Tier

| Руна | Base | Read | Write | Limit |
|------|------|------|-------|-------|
| Spark | +3% damage | — | `(Adjacent, any, Grant(+2% damage))` | ∞ |
| Frost Shard | +4% proj speed | — | `(Ray, any, Grant(+3% proj speed))` | ∞ |
| Vine | +5 HP | — | `(Adjacent, any, Grant(+3 HP))` | ∞ |
| Void Wisp | +2% crit | `(Ring, empty, count) → N` | `(Self, any, Grant(+0.5N% crit))` | 4 |
| Hermit | +10% damage | `(Adjacent, rune, count) → N` | `(Self, any, Grant(10% damage if N == 0 else 0))` | 3 |
| Lens | +1% all stats | — | `(Adjacent, any, Amplify(x1.2))` | 3 |

### Rare Tier

| Руна | Base | Read | Write | Limit |
|------|------|------|-------|-------|
| Inferno Core | +8% damage | — | `(AllSameRune, any, Amplify(x1.15))` | 2 |
| Glacier | +6% slow | — | `(Nearby, any, Grant(+4% slow))` | 2 |
| Living Root | +15 HP | — | `(Cone, any, Grant(regen 1 HP/s))` | 2 |
| Eclipse | +5% crit dmg | — | `(Ring, any, Grant(+3% crit chance))` | 2 |
| Prism | Нет base | `(Adjacent, rune, max(base)) → V` | `(Self, any, Grant(V))` | 2 |
| Radiant Void | +2% damage | `(Adjacent, empty, count) → N` | `(Ring, any, Grant(+2N% damage))` | 2 |
| Nugget | +8% damage | — | — | 2 |

### Epic Tier

| Руна | Base | Read | Write | Limit |
|------|------|------|-------|-------|
| Phoenix Heart | +15% damage | — | `(AllOnGrid, any, Echo)` | 1 |
| Absolute Zero | +10% slow, -5% speed | — | `(Line, any, Suppress(-30%))` | 1 |
| World Tree | +30 HP | `(Nearby, rune, count) → N` | `(Self, any, Grant(+3N HP))` | 1 |
| Singularity | +8% crit | `(Ring, rune, count) → N` | `(Self, any, Grant(+2N% crit))` | 1 |
| Tier Sniper | +5% damage | `(AllOnGrid, tier(Common), count) → N` | `(Self, any, Grant(+1N% damage))` | 1 |

*Radiant Void* — пример cross-pattern: читает пустые ячейки в Adjacent, усиливает руны в Ring пропорционально. *Tier Sniper* — пример Filter по тиру. *Nugget* — pure-base: ни Read, ни Write; даёт высокий base за счёт редкости в пуле и занятого слота.

## Джокеры

Отдельная система поверх сетки — **6 внешних слотов** по периметру гексагона (по одному на каждую сторону: N, NE, SE, S, SW, NW). В слоты кладутся **джокеры** — уникальные руны из отдельного пула, задающие форму билда и меняющие правила (идеология — джокеры Balatro).

```
Схема (слоты * вокруг hex-сетки):

             *
           ╱   ╲
          ╱     ╲
       * ╱ hex   ╲ *
         ╲ сетка ╱
          ╲     ╱
           ╲   ╱
         *       *
             *
```

### Правила

- **Все 6 слотов разблокированы с начала run.** Не покупаются, не прогрессируют.
- **Уникальный пул**, отдельный от рун сетки. Свой список контента, свои механики.
- **Асимметричное взаимодействие с сеткой:**
  - Джокер **читает** сетку (количество рун определённого типа, положение, пустые ячейки, статы игрока, любые условия).
  - Сетка **не видит** джокера. Write-эффекты рун (Grant / Amplify / Suppress / Echo / Chain) не могут выбирать джокера как цель; Read рун не считает джокеров в своих Formulas. Джокер не считается руной сетки.
- **Уникальность.** `Limit=1` на каждого конкретного джокера — двух одинаковых на поле быть не может.
- **Тиры** — те же, что у рун сетки: Common / Rare / Epic / Legendary. Масштаб эффекта растёт с тиром.
- **Никакого общего правила «что умеет джокер»** — как в Balatro, джокер делает что угодно: новую способность, conditional-триггер, рулбук-мод, условный множитель, игнорирование правила, специфическую реакцию на событие.

### Возможные формы эффекта (не исчерпывающий список)

- **Грант способности** — даёт новую активную/пассивную механику (Orbital Shield, Split Shot, Frost Nova и т.п.).
- **Условный множитель** — «+X% damage за каждую Common-руну на сетке».
- **Sector-buff** — «+Y% к base effect рун на ближайшей трети сетки».
- **Рулбук-мод** — «все Adjacent-паттерны теперь работают как Nearby», «Echo рекурсивен до 2 уровней», «первая проданная руна за волну возвращает 100% цены».
- **Event-trigger** — «раз в волну при получении урона: спавнит щит», «при убийстве элиты: +1 reroll в следующем Shop».
- **Meta-влияние на Shop** — «в Shop на 1 предложение больше», «цена реролла −50%».

| Plural | Limit всех рун удваивается | Legendary |

### Экономика джокеров

- **Цена фиксированная на каждого конкретного джокера.** В рамках одного тира разные джокеры могут стоить разное (мощный Rare дороже слабого Rare).
- **Продажа 50%** от цены покупки, освобождает слот.
- В Shop предложениях **перемешаны в общем списке с рунами сетки** (не выделены в отдельную секцию предложения).


При наведении на руну подсвечиваются три множества клеток, каждое своим цветом (пересечения отображаются смешанным):

1. **Что она читает** — ячейки, прошедшие Pattern+Filter её Read.
2. **В кого она пишет** — ячейки, прошедшие Pattern+Filter её Write.
3. **Кто пишет в неё** — руны, чей собственный Write (Pattern+Filter) включает наведённую как цель. Получается обратным обходом: для каждой другой руны проверяем, попадает ли hovered-руна в её Write-зону.

Тултип рядом с курсором показывает:
- Формулу Read с актуальным вычисленным значением (`N = 3`, `V = +6% damage` и т.п.).
- Effect Write с подставленным значением.
- Список рун из множества 3 с краткой расшифровкой их вклада.

При наведении на джокер:
- Показать, что и как он читает с сетки (если есть условия) — подсветить читаемые ячейки/сектор
- Показать текущее значение его эффекта (например, «сейчас активен: +18% damage»)

При перетаскивании руны/джокера из магазина:
- Preview: подсветка паттерна + preview итоговых статов
- Для джокера: подсветка 6 слотов и читаемых зон сетки

Перестановка рун и джокеров в Shop фазе свободная — можно перетаскивать между ячейками/слотами без ограничений.

Во время боя сетка скрыта. Можно посмотреть в режиме паузы (read-only, перестановка запрещена).

## Порядок вычислений

Пайплайн сетки:

1. Собрать base effects всех рун сетки
2. Для каждой руны: **вычислить Read** (если есть) — обойти Pattern с Filter, применить Formula, получить `N`/`V`. Результаты складываются в буфер, чтобы шаг 3 читал согласованный снапшот состояния (никаких race-ов между рунами).
3. Для каждой руны: **применить Write** — в целях Pattern/Filter выполнить Effect, параметризованный значением из Read. Write с `Pattern=Self` пишет в саму руну, Other Write пишет в цели.
4. Применить Echo (Write-эффекты целей срабатывают дважды — рекурсия макс. 1 уровень).
5. Суммировать финальные эффекты сетки → статы игрока.

Джокеры **не привязаны к этому пайплайну** и могут хукаться в любую точку вычислений: до, во время, после, а также на игровые события (вход в волну, получение урона, убийство, покупка в Shop, reroll и т.п.). Каждый джокер сам определяет, в какой момент он срабатывает и что читает — это описывается в его эффекте индивидуально. Write-эффекты рун не выбирают джокеров как цель, Read-Formulas рун не считают джокеров.

**Важно:**
- Echo не рекурсивна (Write-эффект, удвоенный Echo, не удваивается ещё раз). Это предотвращает бесконечные петли.
- Джокеры могут изменять правила пайплайна сетки (например, «Echo срабатывает 3 раза» — влияет на шаг 4), но сами не становятся частью сетки и на них ничто из сетки не действует.

## Экономика

### Получение рун и джокеров
- Каждую Shop фазу предлагается 3-4 случайных позиции из общего пула (руны + джокеры перемешаны, без отдельной секции для джокеров)
- Reroll за gold (цена: TBD)
- Все тиры доступны с самого начала
- **Веса тиров прописаны отдельно для рун и для джокеров.** Вес тира у джокера строго меньше соответствующего тира у руны — джокеры должны выпадать заметно реже рун. Например, если в среднем предложении Shop ~3-4 позиции, джокер встречается не в каждом.
- **В одном предложении Shop не может появиться два одинаковых джокера.** `Limit=1` джокера применяется и к текущему полю, и к прокатываемому предложению: если джокер уже стоит в слоте, он из пула исключён; в рамках одного роллинга предложения дубликаты тоже не допускаются.

### Продажа
- Руну/джокера можно продать обратно за 50% от цены покупки
- Освобождает ячейку сетки или слот джокера

### Разблокировка ячеек
- Старт: Ring 0-1 (7 ячеек)
- Дополнительные ячейки Ring 2 и Ring 3 покупаются за gold
- Цена фиксированная: все ячейки Ring 2 стоят одинаково, все ячейки Ring 3 — одинаково; Ring 3 дороже Ring 2
- Каждая разблокированная ячейка автоматически двигает `rune_limit` (см. Общий лимит рун)

## Мета-прогрессия

Мета между run — **обязательная часть roguelite-контура**. Форма TBD, но рабочая модель:

- **Unlock новых рун и джокеров в пул** — по достижению определённых условий (дойти до Wave N, убить босса, собрать N синергий за run) открываются новые позиции, которые начинают появляться в Shop.
- **Расширяет варианты, не усиливает билд** — мета не делает текущий забег легче, а даёт больше пространства для разных билдов в будущих забегах.
- **Без мета-скидок на старт** — каждый run начинается с одинаковым стартовым капиталом, пустой сеткой 7 ячеек и 6 пустыми слотами джокеров.
