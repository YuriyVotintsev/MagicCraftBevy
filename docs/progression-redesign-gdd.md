# Переработка прогрессии — Game Design Document

## TL;DR

Этот документ фиксирует **перераспределение прогрессии** между двумя контурами:

| Было | Стало |
|------|-------|
| Прогрессия ВНУТРИ забега через Shop с рунами между волнами | Прогрессия ВНУТРИ забега через два контура: (а) модификации карт в колоде на level-up'ах (Balatro-style), (б) реликвии на таймкодах 3/6/9/12/15 мин (in-run аналог джокеров) |
| Между забегами — только unlock нового контента в пул | Между забегами — **полноценная мета-сетка рун** (бывший in-run Shop) |
| Волна → Shop → волна → Shop ... | Непрерывные 30-секундные волны без пауз, враги накапливаются |

Rune Grid (см. `rune-grid-gdd.md`) **как механика сохраняется целиком** — те же руны, те же паттерны, тот же Read/Write. Меняется только **в какой момент игрок взаимодействует с сеткой** — не в Shop-фазе забега, а в мета-экране между забегами.

## Модель забега

### Структура волн

Забег — непрерывная последовательность **волн фиксированной длины**, без пауз между ними.

- **Длина волны:** 30 секунд (стартовое значение, тюнится).
- **Переключение:** по истечении 30 секунд следующая волна стартует **мгновенно**, без экрана/паузы/магазина.
- **Враги НЕ уничтожаются** при смене волны. Всё, что не убито до границы, продолжает жить, двигаться и наносить урон.
- **Спавн** новой волны накладывается поверх недобитых врагов предыдущей.
- **Оси усложнения между волнами:**
  - **Типы врагов** — в первых волнах простые стартеры, позднее подключаются новые типы (tower / spinner / jumper и т.п.).
  - **HP / damage множители** — в волне N все враги получают ×множитель HP и damage (типовой scaling).
  - **Одновременный cap на арене** — растёт максимальное число живых врагов, одновременно присутствующих на арене.

### Смена волны — визуальный маркер

В прототипе **никаких специальных маркеров смены волны нет**. Сверху экрана идёт таймер забега — игрок сам замечает смену состава врагов каждые 30 секунд. Визуальные/аудио стинги — вопрос полировки, не прототипа.

### Конец забега

- **Смерть игрока** → Game Over → возврат в главное меню. Прогрессия уровня и XP текущего забега сбрасываются.
- **Досрочный выход** (quit в меню) — TBD: либо как смерть, либо сохранение результата.
- **Условие победы** на этапе прототипа — **40 волн = 20 минут** (классический Vampire Survivors-хронометраж, босс отложен до появления боссового контента).

## Внутрирунная прогрессия

Внутри забега работают **два параллельных контура роста**:

1. **Колода карт** — постоянный набор стат-карт, который прокачивается через модификации на level-up'ах (Balatro-style: карты + tarot/spectral поверх них). Основной источник количественного роста статов.
2. **Реликвии** — 5 подбираемых за забег эффектов, ломающих правила или открывающих нестандартные взаимодействия. Аналог джокеров из меты, но ограничены рамками одного забега. Основной источник качественного разнообразия билда.

Контуры независимы: выбор модификации на level-up и выбор реликвии на таймкоде — разные экраны с разной логикой.

## Колода карт

Вместо классического survivor-like «выбрал апгрейд из пула — получил» — **модель колоды карт**, идеологически ближе к Balatro: у игрока есть постоянный набор карт, а апгрейды это **модификации** этих карт (аналог tarot / spectral), а не новые сущности.

### Колода и карты

- У игрока есть **колода** — **фиксированный набор из 20 карт** на старте каждого забега. Одна и та же стартовая колода у всех игроков, не редактируется в мете (вариативность приходит через модификации на level-up и через мета-сетку).
- Каждая **карта** — это один стат-бонус игроку: `+1 damage`, `+1 HP`, `+1 crit chance` и т.п.
- Базовые эффекты карт **сознательно очень слабые** — это исходный материал для прокачки, а не готовый билд. Значимый прирост идёт из модификаций.
- Каждая карта имеет **цвет** — один из 4 абстрактных: **Red / Blue / Green / Yellow**. Сам по себе цвет статов не даёт — это носитель, который читается модификациями и синергиями. Абстрактные имена намеренно: никаких скрытых ожиданий «красный = огонь = damage».
- **Распределение в стартовой колоде — равномерное:** 5 карт каждого цвета (5 × 4 = 20). Внутри каждого цвета статы разные — каждый цвет покрывает смешанный набор стат-бонусов, чтобы никакая цветовая стратегия не оказывалась предопределённой с первой секунды.
- **Ничего больше карта изначально не делает:** один маленький стат-бонус + цвет.

### Модификации

Модификации — то, что делает колоду интересной. Навешиваются на уже существующие карты, **не заменяют** их.

Модель слотов на карте — **как в Balatro**: у каждой карты есть несколько **типов-слотов**, в каждом слоте может лежать **максимум одна** модификация своего типа. Зато разные слоты **свободно комбинируются** на одной карте — основной источник мощных карт в поздней стадии забега.

Слоты:

- **Enchant** — усиливает базовый эффект карты (удваивает стат, `+N` к стату, `×1.5` множитель).
- **Seal** — навешивает on-event реакцию (срабатывает дважды, даёт бонус при level-up, при низком HP и т.п.).
- **Stat** — добавляет второй стат к карте (`+1 damage → +1 damage, +1 HP`).
- **Color** — меняет цвет карты (Red → Blue, для пересбора синергии по цветам).
- **Synergy** — условный бонус по состоянию колоды («+X, если в колоде ≥ N красных», «×1.2, если карта третья по счёту цвета»).

**Правила:**
- В каждом слоте максимум одна модификация одновременно. Если предлагается модификация в уже занятый слот — она **заменяет** текущую (переэнчант / пересвап синергии и т.п.). Это валидный и важный вариант выбора: заменить слабый энчант на сильный или переориентировать синергию карты.
- Слоты между собой свободно сочетаются: одна карта может одновременно иметь enchant + seal + stat + color + synergy.
- Конкретный пул модификаций — ориентир ~15-20 позиций, все доступны в прототипе без rarity и без мета-unlock'ов. Градация по силе и rarity вводится после прототипа.

### Level-up и выбор

- Внутри забега **нет валюты** — только XP. XP капается с убийств врагов и автоматически конвертируется в уровни, каждый новый уровень сразу открывает экран выбора модификации. При наборе XP до порога уровня — **полная пауза боя** (как в Vampire Survivors / Brotato). Игрок спокойно читает колоду и выбирает вариант, бой возобновляется после закрытия экрана.
- Экран выбора строится так:
  1. Из колоды **случайно вытягиваются 5 карт** (для первого плейтеста; в дальнейшем число, возможно, уменьшится до 3).
  2. К каждой из пяти **генерируется одна случайная модификация** — в любой из 5 слотов этой карты. Если слот уже занят, предлагаемая модификация **заменит** текущую; это нормальный и валидный вариант.
  3. Игроку показывается **5 вариантов**: «эта конкретная карта + эта конкретная модификация в такой-то слот = такой итоговый эффект».
  4. Игрок выбирает **ровно один** вариант. Модификация применяется к выбранной карте, остальные четыре карты возвращаются в колоду без изменений.
- **Reroll:** на каждом level-up игроку доступен **один бесплатный reroll** текущего предложения. После использования reroll'а генерируется новая пятёрка «карта + мод», кнопка reroll исчезает до следующего уровня. Отказаться от выбора нельзя — после reroll'а (или без него) игрок обязан выбрать один из пяти вариантов.

**Ключевой момент:** игрок не выбирает «какой апгрейд вообще», он выбирает «какую из своих карт прокачать и каким способом». Это делает выбор контекстным — одна и та же модификация на разных картах даёт очень разный результат.

### UI выбора

- **Центр / верх экрана:** 5 вариантов-предложений. Каждый показывает: карту-основу, навешиваемую модификацию, итоговый эффект после применения, изменение стата в понятной формулировке. Если модификация заменяет уже занятый слот — явно подсвечивается «было → стало» для этого слота.
- **Низ экрана / боковая панель:** **вся остальная колода**, одновременно видимая целиком. Для каждой карты видно:
  - цвет,
  - базовый стат и его текущее значение (с учётом уже навешенных модификаций),
  - компактные иконки всех модификаций, которые на ней уже висят.
- Колода должна читаться **беглым взглядом** — без скроллинга, без раскрытия по клику, без «нажмите чтобы посмотреть». Это важнее, чем красивое оформление отдельной карты.
- Опционально: группировка/сортировка колоды по цвету, по стату, по количеству модификаций.
- **Hover-подсветка — максимально помогающая.** Наведение на вариант-предложение подсвечивает в колоде **все карты, с которыми модификация как-либо взаимодействует**: условие синергии (все красные при «+X if ≥N red»), триггеры on-event модификаций, цвет-источник при конвертации цвета и т.п. Идея — игрок не должен считать в голове «сколько у меня сейчас красных», UI честно показывает.

**Почему так:** выбор информированный только если игрок видит контекст. Без видимой колоды и без подсветки связей решения сведутся к «какая модификация звучит жирнее», что обесценивает саму механику.

### Жизненный цикл модификаций

- Все модификации **накапливаются в рамках одного забега**. Одна карта может получить несколько модификаций — по одной в каждый слот (см. раздел «Модификации»), до пяти в сумме.
- При **смерти / конце забега** модификации **полностью сбрасываются**. Колода возвращается в исходное состояние (стартовые карты, никаких модификаций).
- **Стартовая колода** всегда одна и та же (см. раздел «Колода и карты»), модификации к ней не сохраняются между забегами.

## Реликвии

**Реликвии** — параллельный второй контур внутрирунного роста. Это **аналог джокеров** из меты, но ограниченные рамками одного забега: реликвия не усиливает конкретную карту, она **ломает правила** или открывает нестандартное взаимодействие, которое уникализирует билд в этом конкретном забеге.

### Зачем они нужны

Модификации карт дают **количественный рост** — всё те же статы, только сильнее. Реликвии дают **качественное разнообразие**: условные множители, рулбук-моды, on-event триггеры, новые способности. Два забега с одинаковой раскачкой колоды, но разными реликвиями, должны играться заметно по-разному.

### Предложение на таймкоде

- **Каждые 3 минуты (каждые 6 волн)** — игрок получает экран выбора реликвии.
- Всего **5 предложений за забег** на отметках **3 / 6 / 9 / 12 / 15 минут** (волны 6 / 12 / 18 / 24 / 30).
- Последние 10 волн (15 → 20 мин) — **без новых реликвий**. Это «финальный экзамен» на уже собранный билд: реликвий больше не будет, выжимай максимум из того, что есть.
- Экран выбора — **полная пауза боя** (как level-up). Несколько реликвий на выбор, игрок берёт ровно одну. Конкретное число вариантов — TBD (ориентир: 3).
- Отказаться от выбора нельзя. Reroll на этапе прототипа — не даём.

### Чем отличаются от модификаций карт

- **Цель применения:** модификация — конкретная карта в колоде; реликвия — сам забег / глобальное правило.
- **Как часто:** модификация — каждый level-up; реликвия — 5 раз за забег на фиксированных таймкодах.
- **Тип эффекта:** модификация — усиление существующих статов; реликвия — новое правило / нестандартное взаимодействие.
- **Взаимодействие с колодой:** модификация живёт внутри карты; реликвия может читать состояние колоды, но сама не часть её.

### Примеры реликвий

Конкретный пул — TBD, но форма эффектов такая:

- «Все модификации Synergy на твоих картах удваиваются».
- «Каждое 5-е убийство триггерит взрыв».
- «Каждая карта считается дважды в условных проверках `≥ N красных` и т.п.».
- «-1 свободный выбор на level-up, но модификации применяются в два слота сразу».
- «Добавить в колоду одну случайную карту».
- «Пока HP ≤ 20%: все карты дают удвоенный эффект».
- «Synergy-условия по цвету читают также соседний цвет (red считает blue как red)».
- «При выборе модификации: доступен один дополнительный reroll на этом уровне».

Главное свойство — **не плоский стат-бафф**. Любая реликвия должна заставлять игрока играть чуть иначе или пересобирать план, а не просто увеличивать числа.

### Жизненный цикл реликвий

- Накапливаются в рамках одного забега. Максимум — 5 штук к финалу.
- Пул реликвий **отдельный**: не пересекается с рунами мета-сетки и с модификациями карт.
- При конце забега (смерть или победа) **все реликвии сбрасываются**. В мету не уходят, не копятся.

## Связь контуров

Три контура прогрессии, связь между ними:

- **Мета-сетка рун** (между забегами) — задаёт **стартовые статы** игрока на забег. Применяется один раз при старте.
- **Колода карт** (в забеге, level-up'ы) — задаёт **количественный рост** статов через модификации.
- **Реликвии** (в забеге, таймкоды) — задают **качественное разнообразие** билда через нестандартные правила.

Все три складываются аддитивно: стартовая база из сетки + прокачанная колода + активные реликвии. Колода и реликвии независимы друг от друга (реликвия может читать состояние колоды, но колода не меняется от реликвии).

## Мета-прогрессия — Rune Grid

Rune Grid переезжает из внутризабеговой Shop-фазы в **мета-контур между забегами**.

### Где живёт сетка

- **Экран мета-прогрессии** доступен из главного меню (до старта забега).
- Сетка, джокерские слоты, пул разблокированных ячеек — **персистентное состояние** (сохраняется между забегами и между сессиями).
- Игрок свободно редактирует сетку в мета-экране: drag-drop рун, swap-ы, продажа, покупка в Shop — всё, что раньше было в Shop-фазе забега.

### Эффект сетки на забег

- Стартовые статы игрока в забеге = **итог работы всех Read/Write-эффектов** текущей мета-сетки.
- Сетка применяется **один раз при старте забега** и фиксирует стартовый набор модификаторов. Во время забега сетка не пересчитывается — забег идёт с замороженным результатом.
- Джокеры из мета-сетки применяются на старте наравне с рунами (в прототипе — в текущем виде, см. раздел «Джокеры»).

### Экономика мета (прототип)

Для прототипа экономика мета сведена к минимуму:

- **Мета-валюта:** одна, условные «мета-монеты».
- **Начисление:** **100 монет за победный забег** (дошёл до 40 волн), **10 монет за проигранный** (любой забег, закончившийся смертью или quit'ом). Ничего пропорционального глубине — плоские фиксы, не тюним на этом этапе.
- **Мета-Shop** (покупка рун, разблокировка ячеек сетки, reroll и т.п.) **в этом документе не описывается и не трогается** — используется текущая реализация как есть. Мета-монеты просто становятся его валютой. Любая переработка мета-Shop'а — отдельная задача.
- **Unlock новых рун в пул** — в прототипе НЕ вводим, все руны доступны в мета-Shop сразу. Unlock-ветка — после прототипа.

### Джокеры (прототип)

В прототипе — **не трогаем**. Остаются как в текущей реализации («жёлтые шарики» в слотах), их роль и поведение пересматриваются отдельной задачей после прототипа. В этом документе их присутствие/отсутствие не обсуждаем.

### Пресеты сетки

Один персистентный стейт сетки. Никаких build slots / сохранённых билдов — игрок редактирует одну сетку между забегами и играет с ней.

### Permadeath — что значит сейчас

В старой модели permadeath сбрасывал **всю сетку**. В новой — сетка **мета**, то есть **не сбрасывается**. Permadeath теперь означает:

- Сброс уровня и XP текущего забега.
- Сброс всех модификаций, навешанных на карты колоды за забег. Колода возвращается к стартовому виду.
- Сброс всех собранных за забег реликвий.
- Начисление мета-монет по итогу забега (10 за проигранный, 100 за победный — см. раздел «Экономика мета»). Никакого частичного трансфера по глубине.
- **Сетка рун, джокеры мета, разблокированные ячейки, мета-монеты — сохраняются.**

Это **сдвигает жанр** обратно ближе к incremental/meta-heavy roguelite (ориентиры: Halls of Torment, Death Must Die), а не чистый Brotato, где всё сбрасывается.

## Открытые вопросы

- [ ] Конкретные кривые усложнения волн (HP/damage множитель по волнам, одновременный cap врагов на арене по волнам, момент подключения каждого типа врагов). Валидируется playtest'ом.
- [ ] Баланс мета vs in-run: стартовые статы от мета-сетки не должны «перекрывать» рост через колоду, и наоборот — прокачка через модификации в рамках одного забега не должна делать мета-сетку бессмысленной. Валидируется playtest'ом.
- [ ] Размер пула реликвий для прототипа (ориентир 10-15).
- [ ] Сколько реликвий показывать в одном окне выбора: 3 (как на level-up до обсуждения), 5 (как сейчас на level-up), или другое.
- [ ] Взаимодействие реликвий с модификациями карт: разрешаем ли реликвии, напрямую читающие / модифицирующие модификации (например, «удвой все Synergy»), или держим границу между контурами жёстче.
- [ ] Можно ли по ходу забега получить одну и ту же реликвию дважды (дубль на поле), или `Limit=1` на каждую.
